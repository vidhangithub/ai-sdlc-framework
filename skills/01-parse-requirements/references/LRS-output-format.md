# LRS Output Format Reference

Use this exact structure when producing a Locked Requirement Specification.

---

## Document Header

```markdown
# Locked Requirement Specification

| Field | Value |
|-------|-------|
| LRS-ID | LRS-{YYYYMMDD}-{001} |
| Status | DRAFT |
| Version | 1.0.0 |
| Source | {link or description of source artifact} |
| Service | {service-name} |
| Generated | {date} |
| Generated By | AI (parse-requirements skill v1.0.0) |
| Reviewed By | — (pending) |
```

---

## Section 1: Business Context

```markdown
## 1. Business Context

| Field | Value |
|-------|-------|
| Business Objective | {1-2 sentences — what business problem does this solve} |
| Primary User / Persona | {who uses this system directly} |
| Business Value | {measurable business outcome expected} |
| Success Metric | {how will we know this succeeded} |
| Target Delivery | {sprint / quarter if mentioned} |
| Priority | HIGH / MEDIUM / LOW |
```

---

## Section 2: Functional Requirements

```markdown
## 2. Functional Requirements

| ID | Requirement | Priority | Testable? | Notes |
|----|------------|----------|-----------|-------|
| LRS-{date}-FR-01 | The system SHALL {verb} {object} when {condition} | MUST | YES | |
| LRS-{date}-FR-02 | The system SHALL {verb} | MUST | YES | [ASSUMED: ...] |
| LRS-{date}-FR-03 | The system SHOULD {verb} | SHOULD | YES | |
```

**Priority values:** MUST (mandatory) | SHOULD (recommended) | COULD (nice-to-have)

---

## Section 3: Non-Functional Requirements

```markdown
## 3. Non-Functional Requirements

| Category | Requirement | Metric / Threshold | Source |
|----------|------------|-------------------|--------|
| Performance | API response time | P95 < 500ms at 100 TPS | [ASSUMED] |
| Availability | Service uptime | 99.9% monthly | Stated |
| Scalability | Peak load handling | 10x baseline without degradation | [ASSUMED] |
| Security | Auth mechanism | OAuth2 / OIDC | Stated |
| Data Retention | Audit records | 7 years minimum | [ASSUMED - regulatory] |
| DR / RTO | Recovery time | < 1 hour | [ASSUMED] |
| DR / RPO | Recovery point | < 15 minutes | [ASSUMED] |
```
