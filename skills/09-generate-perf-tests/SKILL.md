# Skill: Generate Gatling Performance Tests

## Purpose

Produce a production-grade Gatling 3.x simulation that validates the service against its LRS-defined NFRs under realistic and extreme load conditions, with CI-enforceable SLA assertions that fail the build when thresholds are breached.

---

## Step-by-Step Instructions

### Step 1 — Extract SLAs from LRS NFR Section

Parse the LRS NFR section and extract:
- **Response time SLA:** P95 threshold in milliseconds
- **Throughput target:** TPS (transactions per second)
- **Availability SLA:** Acceptable error rate
- **Scalability requirement:** Peak load multiplier

If any are missing, apply these conservative defaults and flag them:
```
[ASSUMED] P95 < 500ms — confirm with architect
[ASSUMED] Baseline: 10 TPS, Peak: 100 TPS — confirm with architect
[ASSUMED] Error rate < 1% — confirm with architect
```

### Step 2 — Define Scenario Configuration

Produce a configuration block at the top of the simulation:

```scala
// ── SLA CONFIGURATION (derived from LRS NFRs) ─────────────────
object SLAConfig {
  // Response time thresholds
  val p95ResponseTimeMs = 500       // LRS NFR: P95 < 500ms
  val p99ResponseTimeMs = 1000      // [ASSUMED] P99 < 1000ms

  // Load profile
  val baselineUsers    = 10
  val normalLoadUsers  = 50
  val peakLoadUsers    = 100
  val spikeUsers       = 200

  // Duration
  val rampDuration     = 30         // seconds
  val sustainDuration  = 300        // seconds (5 minutes)

  // Acceptable error rate (0.01 = 1%)
  val maxErrorRate     = 0.01

  // Target environment
  val baseUrl = "{target_env}"
}
```

### Step 3 — Write the Complete Simulation

Generate the full Gatling simulation following this structure:

```scala
// File: src/gatling/scala/{ServiceName}Simulation.scala
// LRS Reference: {LRS-ID}
// Generated by: generate-perf-tests skill v1.0.0

package com.company.perf

import io.gatling.core.Predef._
import io.gatling.http.Predef._
import scala.concurrent.duration._
import scala.util.Random

class {ServiceName}Simulation extends Simulation {

  // ── HTTP PROTOCOL ──────────────────────────────────────────
  val httpProtocol = http
    .baseUrl(SLAConfig.baseUrl)
    .acceptHeader("application/json")
    .contentTypeHeader("application/json")
    .header("X-Correlation-Id", session => java.util.UUID.randomUUID().toString)
    .shareConnections

  // ── AUTH TOKEN (fetched once, shared across users) ─────────
  // Option A: Static token (for environments with long-lived test tokens)
  val authToken = sys.env.getOrElse("PERF_AUTH_TOKEN",
    throw new IllegalStateException("PERF_AUTH_TOKEN env var required"))

  // Option B: Dynamic OAuth2 token (uncomment if token URL provided)
  // val authToken = {
  //   val response = http("Fetch Auth Token")
  //     .post("{token_url}")
  //     .formParam("grant_type", "client_credentials")
  //     .formParam("client_id", sys.env("PERF_CLIENT_ID"))
  //     .formParam("client_secret", sys.env("PERF_CLIENT_SECRET"))
  //   // Parse token from response — implement as setUp exec step
  // }

  // ── TEST DATA FEEDERS ──────────────────────────────────────
  val requestFeeder = csv("test-data/{service-name}-requests.csv").circular

  // Random UUID generator for path params
  def randomId(): String = java.util.UUID.randomUUID().toString

  // ── SCENARIOS ──────────────────────────────────────────────

  // SCENARIO 1: Create {resource}
  val createScenario = scenario("Create {Resource}")
    .feed(requestFeeder)
    .exec(
      http("POST /api/v1/{resource}")
        .post("/api/v1/{resource}")
        .header("Authorization", s"Bearer ${authToken}")
        .header("X-Idempotency-Key", session => randomId())
        .body(StringBody(session =>
          s"""
            |{
            |  "reference": "${session("reference").as[String]}",
            |  "amount": ${session("amount").as[String]},
            |  "currency": "${session("currency").as[String]}"
            |}
          """.stripMargin
        )).asJson
        .check(status.is(201))
        .check(jsonPath("$.id").exists.saveAs("createdId"))
        .check(responseTimeInMillis.lte(SLAConfig.p95ResponseTimeMs * 2))
    )
    .pause(100.milliseconds, 500.milliseconds) // Think time between requests

  // SCENARIO 2: Get {resource} by ID
  val getScenario = scenario("Get {Resource} by ID")
    .exec(
      http("GET /api/v1/{resource}/{id}")
        .get(s"/api/v1/{resource}/${randomId()}")
        .header("Authorization", s"Bearer ${authToken}")
        .check(status.in(200, 404)) // 404 acceptable for random IDs
        .check(responseTimeInMillis.lte(SLAConfig.p95ResponseTimeMs))
    )
    .pause(50.milliseconds, 200.milliseconds)

  // SCENARIO 3: Mixed read/write (realistic production mix)
  val mixedScenario = scenario("Mixed Read/Write")
    .feed(requestFeeder)
    .exec(
      http("POST /api/v1/{resource} [mixed]")
        .post("/api/v1/{resource}")
        .header("Authorization", s"Bearer ${authToken}")
        .header("X-Idempotency-Key", session => randomId())
        .body(StringBody(session =>
          s"""{"reference": "${session("reference").as[String]}", "amount": 100.00, "currency": "GBP"}"""
        )).asJson
        .check(status.is(201))
        .check(jsonPath("$.id").saveAs("newId"))
    )
    .pause(200.milliseconds)
    .exec(
      http("GET /api/v1/{resource}/{id} [mixed]")
        .get("/api/v1/{resource}/#{newId}")
        .header("Authorization", s"Bearer ${authToken}")
        .check(status.is(200))
    )

  // ── LOAD PROFILES ──────────────────────────────────────────

  // PROFILE 1: Baseline — smoke test the API under light load
  val baselineInjection = Seq(
    rampUsers(SLAConfig.baselineUsers).during(SLAConfig.rampDuration.seconds),
    constantUsersPerSec(SLAConfig.baselineUsers / 10.0)
      .during(SLAConfig.sustainDuration.seconds)
  )

  // PROFILE 2: Normal Load
  val normalLoadInjection = Seq(
    rampUsers(SLAConfig.normalLoadUsers).during(SLAConfig.rampDuration.seconds),
    constantUsersPerSec(SLAConfig.normalLoadUsers / 10.0)
      .during(SLAConfig.sustainDuration.seconds)
  )

  // PROFILE 3: Peak Load
  val peakLoadInjection = Seq(
    rampUsers(SLAConfig.peakLoadUsers / 2).during(30.seconds),
    rampUsers(SLAConfig.peakLoadUsers).during(30.seconds),
    constantUsersPerSec(SLAConfig.peakLoadUsers / 10.0)
      .during(SLAConfig.sustainDuration.seconds)
  )

  // PROFILE 4: Spike — sudden traffic surge
  val spikeInjection = Seq(
    constantUsersPerSec(5).during(30.seconds),          // Baseline
    rampUsers(SLAConfig.spikeUsers).during(10.seconds),  // Spike up
    constantUsersPerSec(SLAConfig.spikeUsers / 10.0)
      .during(60.seconds),                              // Sustain spike
    rampUsers(5).during(10.seconds),                    // Spike down
    constantUsersPerSec(5).during(30.seconds)           // Recover
  )

  // ── SLA ASSERTIONS ─────────────────────────────────────────
  val slaAssertions = Seq(
    // Response time assertions
    global.responseTime.percentile(95).lte(SLAConfig.p95ResponseTimeMs),
    global.responseTime.percentile(99).lte(SLAConfig.p99ResponseTimeMs),
    global.responseTime.mean.lte(SLAConfig.p95ResponseTimeMs / 2),

    // Success rate assertion
    global.successfulRequests.percent.gte(
      ((1 - SLAConfig.maxErrorRate) * 100).toInt
    ),

    // Request count sanity check
    global.requestsPerSec.gte(1.0),

    // Per-request-type assertions
    details("POST /api/v1/{resource}")
      .responseTime.percentile(95).lte(SLAConfig.p95ResponseTimeMs),
    details("GET /api/v1/{resource}/{id}")
      .responseTime.percentile(95).lte(SLAConfig.p95ResponseTimeMs / 2)
  )

  // ── SETUP ──────────────────────────────────────────────────
  // Select profile based on system property: -Dscenario=baseline|normal|peak|spike
  val selectedScenario = sys.props.getOrElse("scenario", "normal")

  val injection = selectedScenario match {
    case "baseline" => createScenario.inject(baselineInjection: _*)
    case "peak"     => mixedScenario.inject(peakLoadInjection: _*)
    case "spike"    => createScenario.inject(spikeInjection: _*)
    case _          => mixedScenario.inject(normalLoadInjection: _*) // default: normal
  }

  setUp(injection.protocols(httpProtocol))
    .assertions(slaAssertions: _*)
}
```

### Step 4 — Generate Test Data Feeder CSV

```csv
reference,amount,currency
REF-001,100.00,GBP
REF-002,250.50,GBP
REF-003,999.99,USD
REF-004,1500.00,EUR
REF-005,75.25,GBP
```

Save as: `src/gatling/resources/test-data/{service-name}-requests.csv`

### Step 5 — Generate Maven/Gradle Config

```xml
<!-- pom.xml — Gatling Maven plugin -->
<plugin>
  <groupId>io.gatling</groupId>
  <artifactId>gatling-maven-plugin</artifactId>
  <version>4.9.0</version>
  <configuration>
    <simulationClass>com.company.perf.{ServiceName}Simulation</simulationClass>
    <resultsFolder>${project.build.directory}/gatling</resultsFolder>
    <runDescription>Performance test — {service-name}</runDescription>
  </configuration>
</plugin>
```

**Run commands:**
```bash
# Baseline (CI on every release branch)
mvn gatling:test -Dscenario=baseline

# Normal load (pre-UAT)
mvn gatling:test -Dscenario=normal

# Peak load (performance environment only)
mvn gatling:test -Dscenario=peak -Denv=perf

# Spike (performance environment only)
mvn gatling:test -Dscenario=spike -Denv=perf

# View report
open target/gatling/{ServiceName}Simulation-*/index.html
```

## Quality Checklist

- [ ] All SLAs derived from LRS NFR section (or flagged as assumed)
- [ ] All 4 load profiles present: baseline, normal, peak, spike
- [ ] SLA assertions present and configured to fail the build
- [ ] Auth token configured via environment variable (not hardcoded)
- [ ] Test data feeder CSV generated and referenced
- [ ] Think time (pause) between requests to simulate real users
- [ ] Mixed read/write scenario reflects production traffic pattern
- [ ] Per-endpoint SLA assertions in addition to global assertions
- [ ] Run commands documented for each scenario
- [ ] Correlation ID header included in all requests

---

## Rules

- **Never hardcode auth tokens** — always use environment variables
- **Never hardcode target URLs** — parameterise via system property or env var
- **SLA assertions must cause build failure** — use `.assertions()` not just reports
- **Always include think time** — zero pause is not realistic user behaviour
- **Always include a mixed scenario** — pure write or pure read is not representative
